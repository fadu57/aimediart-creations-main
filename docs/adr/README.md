# ADR — Architecture Decision Records

Chaque décision structurante est consignée ici en note courte, conformément à
`engineering-standards` : une décision = un fichier `NNNN-titre-court.md`.

Format :

```markdown
# NNNN — Titre de la décision

Date : YYYY-MM-DD · Statut : proposée | acceptée | remplacée par NNNN

## Contexte
Le problème et les contraintes, en quelques phrases.

## Options envisagées
2–3 options avec leur trade-off principal.

## Décision
L'option retenue et pourquoi.

## Conséquences
Ce que cela implique (positif et négatif), et ce qui devient plus difficile.
```

Les post-mortems d'incident mettent à jour les standards d'ingénierie.
